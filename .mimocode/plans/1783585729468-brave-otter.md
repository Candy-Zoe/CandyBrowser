# CandyBrowser 全面改进计划

## 现状分析

这是一个 WPF + WebView2 (.NET 8) 浏览器项目，架构合理（MVVM + DI + EF Core + SQLite），但存在多个关键功能缺陷，与 Edge 差距明显。

---

## P0 - 核心功能修复（必须先做）

### 1. 多标签页真正实现

**问题**：`MainWindow.xaml:152` 只有一个 `BrowserViewControl`，切换标签页只是对同一个 WebView2 重新导航，所有标签共享一个页面状态。

**方案**：
- 在 `MainViewModel` 中维护一个 `Dictionary<long, BrowserView>` 映射 tab ID → WebView2 实例
- 修改 `MainWindow.xaml`：用 `ContentControl` + `DataTemplate` 替换固定的 `BrowserViewControl`，或在代码中动态创建/隐藏 WebView2
- 切换标签时：隐藏当前 WebView2，显示目标 WebView2（不是重新导航）
- 新建标签时：创建新的 `BrowserView` 实例并初始化 WebView2
- 关闭标签时：Dispose 对应的 WebView2 并从字典移除

**修改文件**：
- `src/CandyBrowser.Windows/MainWindow.xaml` - 将 `BrowserView` 改为动态容器
- `src/CandyBrowser.Windows/MainWindow.xaml.cs` - 添加 Tab↔WebView2 映射管理
- `src/CandyBrowser.Windows/ViewModels/MainViewModel.cs` - Tab 切换逻辑改为显示/隐藏

## P1 - 重要体验改进

### 2. 地址栏增强：搜索建议 + 自动补全

**问题**：`SuggestionItem` 模型已存在但从未使用，地址栏只是一个纯 TextBox。

**方案**：
- 在地址栏下方添加一个 `Popup` + `ListBox` 显示搜索建议
- 输入时异步查询：历史记录匹配 + 书签匹配 + 搜索引擎建议
- 用 `IHistoryService.SearchAsync()` 和 `IBookmarkService.SearchAsync()` 获取建议
- 支持上下键选择、Enter 确认、Esc 关闭
- 去掉 `AddressBar_LostFocus` 中重置 URL 的行为（会丢失用户输入）

**修改文件**：
- `src/CandyBrowser.Windows/MainWindow.xaml` - 地址栏增加 Popup
- `src/CandyBrowser.Windows/MainWindow.xaml.cs` - 搜索建议逻辑
- `src/CandyBrowser.Windows/ViewModels/MainViewModel.cs` - 添加搜索建议相关属性和命令

### 3. UI 美化：图标 + 视觉层次

**问题**：工具栏按钮使用纯文字（←、→、R、Home），没有视觉吸引力。

**方案**：
- 使用 Unicode 符号或 Path 数据替代纯文字按钮：
  - 后退: `⌂` 或 Path ←
  - 前进: `⌂` 或 Path →
  - 刷新: `↻`
  - 主页: `⌂`
  - 书签: `☆` / `★`
  - 设置: `⚙`
- 地址栏添加圆角和 padding
- 标签页样式改进：选中态更明显、关闭按钮 hover 效果
- 工具栏增加适当间距

**修改文件**：
- `src/CandyBrowser.Windows/MainWindow.xaml` - 全面 UI 改进
- `src/CandyBrowser.Windows/Themes/LightTheme.xaml` - 添加新样式

### 4. 搜索引擎设置实际生效

**问题**：`MainViewModel.NavigateToUrl()` 中硬编码了 `https://www.bing.com/search?q=`，没有读取用户设置的搜索引擎。

**方案**：
- 在 `NavigateToUrl()` 中从 `_settingsService.GetSearchEngineAsync()` 获取搜索引擎模板
- 用获取的模板替换硬编码的 Bing

**修改文件**：
- `src/CandyBrowser.Windows/ViewModels/MainViewModel.cs` - 读取搜索引擎设置

## P2 - 功能完善

### 5. 键盘快捷键完善

**问题**：只有 Ctrl+T 和 F5。

**方案**：在 `MainWindow.xaml` 添加：
- `Ctrl+W` - 关闭当前标签
- `Ctrl+L` / `F6` - 聚焦地址栏
- `Ctrl+D` - 添加书签
- `Alt+Left` - 后退
- `Alt+Right` - 前进
- `Ctrl+Tab` - 下一个标签
- `Ctrl+Shift+Tab` - 上一个标签
- `Ctrl+1~9` - 跳转到第 N 个标签
- `F12` - 开发者工具
- `Ctrl+J` - 打开下载管理器

**修改文件**：
- `src/CandyBrowser.Windows/MainWindow.xaml` - InputBindings
- `src/CandyBrowser.Windows/ViewModels/MainViewModel.cs` - 对应命令

### 6. 书签栏刷新 + 添加书签后更新

**问题**：添加书签后书签栏不刷新，`AddBookmarkAsync` 完成后没有调用 `RefreshBookmarksBar()`。

**方案**：
- 在 `MainViewModel.AddBookmarkAsync()` 完成后，通过事件或直接调用通知 MainWindow 刷新书签栏
- 在 `MainViewModel` 中添加 `event EventHandler? BookmarkChanged` 事件
- `MainWindow` 订阅此事件并调用 `RefreshBookmarksBar()`

**修改文件**：
- `src/CandyBrowser.Windows/ViewModels/MainViewModel.cs` - 添加 BookmarkChanged 事件
- `src/CandyBrowser.Windows/MainWindow.xaml.cs` - 订阅事件并刷新

### 7. 新标签页页

**问题**：新标签页就是空白页。

**方案**：创建一个 `NewTabPage.xaml` UserControl，包含：
- 搜索框（居中大搜索框）
- 快速链接（从书签栏取前 8 个常用书签）
- 最近访问的网站（从历史记录取）
- 当前日期和问候语

在新建标签时，先显示这个 NewTabPage，用户输入 URL 后再切换到 WebView2。

**修改文件**：
- 新建 `src/CandyBrowser.Windows/Views/NewTabPage.xaml` + `.cs`
- `src/CandyBrowser.Windows/MainWindow.xaml` - 支持显示 NewTabPage
- `src/CandyBrowser.Windows/MainWindow.xaml.cs` - 标签切换逻辑

### 8. 设置实时生效

**问题**：字体大小、书签栏可见性等设置保存后不会实时应用。

**方案**：
- 保存外观设置后，通过事件通知主窗口应用更改
- 字体大小 → 通过 WebView2 的 `ZoomFactor` 或注入 CSS 实现
- 书签栏可见性 → 直接控制 `BookmarksBarBorder.Visibility`
- 状态栏可见性 → 直接控制 BrowserView 中状态栏 Visibility

**修改文件**：
- `src/CandyBrowser.Windows/Views/SettingsWindow.xaml.cs` - 保存后触发事件
- `src/CandyBrowser.Windows/MainWindow.xaml.cs` - 响应设置变更

**方案**：在 BrowserView.xaml 的 ContextMenu 中添加：

**修改文件**：
- `src/CandyBrowser.Windows/Views/BrowserView.xaml` - 增强右键菜单
- `src/CandyBrowser.Windows/Views/BrowserView.xaml.cs` - 实现 OpenInNewTab
