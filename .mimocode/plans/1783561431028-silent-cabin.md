# CandyZoe浏览器 - Implementation Plan

## Overview

Build a cross-platform browser (CandyZoe浏览器) with Windows Desktop (WPF + .NET 8 + WebView2) and Android (Kotlin + Jetpack Compose + Android WebView). The project starts from an empty git repo. All core Edge-like features are required: multi-tab, address bar, bookmarks, history, PDF reader, extensions, reading mode, password manager, DevTools.

## Phased Implementation Plan

### Phase 1: Foundation (Weeks 1-3) — Milestone: "It browses"

#### Week 1: Solution Scaffold + Data Layer

**Create:**
- `CandyBrowser.sln` — solution file referencing all projects
- `Directory.Build.props` — shared MSBuild properties (version, analyzers)
- `.gitignore` — .NET + Android + IDE ignores
- `.editorconfig` — code style rules

**Create `CandyBrowser.Core`:**
- `CandyBrowser.Core.csproj` (net8.0 class library)
- `Models/Bookmark.cs` — Bookmark model (Id, ParentId, Title, Url, FaviconUrl, Position, IsFolder, CreatedAt, UpdatedAt, SyncId)
- `Models/HistoryEntry.cs` — History model (Id, Url, Title, FaviconUrl, VisitCount, LastVisit, FirstVisit, DurationMs, SyncId)
- `Models/PasswordEntry.cs` — Password model (Id, Domain, Username, Password, Url, CreatedAt, UpdatedAt, SyncId)
- `Models/TabInfo.cs` — Tab model (Id, WindowId, Position, Url, Title, FaviconUrl, IsPinned, IsIncognito, ParentId, CreatedAt, UpdatedAt)
- `Models/ExtensionInfo.cs` — Extension model (Id, ExtensionId, Name, Version, Description, ManifestJson, InstallPath, IsEnabled, Permissions, IconPath, InstalledAt, UpdatedAt)
- `Models/Setting.cs` — Setting model (Key, Value, ValueType, UpdatedAt)
- `Models/SuggestionItem.cs` — Address bar suggestion (Type, Title, Url)
- `Enums/SuggestionType.cs` — Bookmark, History, Search, Url
- `Enums/ReadingTheme.cs` — Light, Dark, Sepia

**Create `CandyBrowser.Shared.Abstractions`:**
- `CandyBrowser.Shared.Abstractions.csproj` (netstandard2.0)
- `INavigationService.cs` — Navigate, GoBack, GoForward, Reload, Stop, GoHome + events
- `IBookmarkService.cs` — CRUD, Search, GetTree, GetAll
- `IHistoryService.cs` — Add, Search, Delete, Clear, GetAll
- `IPasswordService.cs` — Save, Get, Delete, Search, Encrypt/Decrypt
- `IExtensionService.cs` — Install, Uninstall, Enable, Disable, GetAll, GetManifest
- `ITabManager.cs` — Create, Close, Switch, GetAll, SaveState, RestoreState
- `ISettingsService.cs` — Get, Set, GetSearchEngine, GetHomepage
- `IPdfService.cs` — OpenPdf, ExtractText
- `IReadingModeService.cs` — ExtractContent, RenderAsCleanHtml

**Create `CandyBrowser.Data`:**
- `CandyBrowser.Data.csproj` (net8.0, references EF Core SQLite 8.0)
- `Contexts/CandyBrowserDbContext.cs` — DbContext with DbSets for all entities
- `Entities/BookmarkEntity.cs` — EF Core entity matching bookmark schema
- `Entities/HistoryEntity.cs`

- `Entities/TabEntity.cs`

- `Configurations/HistoryEntityConfiguration.cs`

- `ViewModels/MainViewModel.cs` — Tab management, active tab tracking
- `ViewModels/BrowserTabViewModel.cs` — Per-tab state (URL, title, isLoading, canGoBack/Forward)
- `ViewModels/AddressBarViewModel.cs` — URL input, suggestion generation
- `Services/WebView2Manager.cs` — CoreWebView2 environment init + profile management
- `Services/WpfNavigationService.cs` — INavigationService impl wrapping WebView2
- `Services/WpfHistoryService.cs` — IHistoryService impl (record visits on NavigationCompleted)

#### Week 3: Navigation + History + Tab Management

- Wire up address bar → navigation flow (type URL → Enter → navigate)
- Smart URL detection (is it a URL or search query?)
- Search engine integration (configurable, default: Bing/Baidu)
- History recording on NavigationCompleted
- Tab creation (New Tab button) + closing (X button) + switching (click tab)
- Keyboard shortcuts: Ctrl+T (new tab), Ctrl+W (close tab), Ctrl+Tab (next tab), Ctrl+L (focus address bar)

### Phase 2: Core Features (Weeks 4-6) — Milestone: "It's useful"

#### Week 4-5: Bookmarks + History Panel

- Bookmark sidebar with tree view (folders + items)
- Add bookmark button (star icon in address bar)
- Bookmark CRUD: add, edit title/url, delete, move between folders
- History panel with search, date grouping, delete single/all
- Address bar autocomplete: query bookmarks + history + search engine suggestions

#### Week 6: Settings + Theme + Session

- Settings window: homepage, search engine, theme (light/dark/system), font size, startup behavior
- Theme switching (light ↔ dark) with ResourceDictionary swap
- Session restore: save open tabs on close, restore on launch
- Privacy settings: clear browsing data, clear history, clear passwords

---

### Phase 3: Power Features (Weeks 7-9) — Milestone: "It's powerful"

#### Week 7: PDF + Reading Mode

- PDF: WebView2 natively renders .pdf URLs — no extra code needed
- Reading mode: Extract page HTML via `ExecuteScriptAsync`, parse with HtmlAgilityPack, render in clean WebView2 tab
- Reading mode themes (light/dark/sepia), font size control

#### Week 8: Password Manager

- Save password prompt on form submission (detect password fields via JS injection)
- AES-256-GCM encryption for stored passwords (master password or Windows Credential Manager)
- Password manager window: list, search, show/hide, copy, autofill
- Password generator (configurable length, character types)

#### Week 9: DevTools + Extension Scaffold

- DevTools: `webView.OpenDevToolsWindow()` — one line, full Chrome DevTools
- DevTools docked panel option: split view with bottom panel
- Extension system scaffold: manifest.json parser, extension directory scanner
- Extension manager window: list installed, enable/disable toggle

---

### Phase 4: Extensions & Polish (Weeks 10-12) — Milestone: "It's extensible"

#### Week 10-11: Extension Loading

- Load unpacked extensions via `CoreWebView2Profile.AddBrowserExtension()`
- Extension popup rendering (new Window with WebView2)
- Content script injection on page load
- Permission system: runtime prompts for sensitive APIs
- Install .crx files (extract, validate manifest, register)

#### Week 12: Polish

- Loading indicator in tab
- Error pages (connection failed, DNS error)
- Context menus (right-click: open in new tab, copy link, save as)
- Download manager
- Print dialog
- Keyboard shortcuts completion (F5 refresh, F11 fullscreen, Ctrl+F find in page)

---

### Phase 5: Android (Weeks 13-18) — Milestone: "It's cross-platform"

#### Week 13: Android Scaffold

- `CandyBrowser.Android/` with `app/build.gradle.kts` (minSdk 26, targetSdk 34)
- Dependencies: Compose BOM, Material 3, Room, Hilt, Navigation Compose, WebKit

- Hilt modules: `AppModule.kt`, `DatabaseModule.kt`, `RepositoryModule.kt`

#### Week 14-15: Android Browser Core

- `BrowserScreen.kt` — Main screen with WebView + toolbar
- `AddressBar.kt` — URL input with autocomplete (Compose)
- `TabStrip.kt` — Horizontal scrollable tab chips
- `NavigationBar.kt` — Back/Forward/Refresh/Home icons
- `BrowserViewModel.kt` — Tab management, navigation state
- `NavigationService.kt` — Android WebView wrapper
- Bookmark and History screens with CRUD

#### Week 16-17: Android Power Features

- Reading mode (JSoup content extraction, clean WebView render)
- PDF viewer (PDF.js in WebView)
- Password manager (Room + encryption)
- Extension system (JS bridge via `addJavascriptInterface`)
- Settings screen

#### Week 18: Android Polish + Release

- Material 3 theming (light/dark)
- Bottom sheet for tab overview
- Swipe gestures (swipe back/forward)
- Share integration (share URL to browser)
- Performance optimization, testing, release prep

## Key Technology Choices

| Component | Technology | Rationale |
|-----------|-----------|-----------|
| Desktop UI | WPF + .NET 8 | Mature, good WebView2 support, MVVM ecosystem |
| Rendering Engine | WebView2 (Edge Chromium) | Chromium rendering, Chrome extension compat, built-in PDF |
| MVVM Toolkit | CommunityToolkit.Mvvm | Source generators, minimal boilerplate |
| DI Container | Microsoft.Extensions.DependencyInjection | Standard .NET |
| Desktop ORM | EF Core + SQLite | LINQ, migrations, well-documented |
| Android UI | Jetpack Compose + Material 3 | Modern declarative UI |
| Android DI | Hilt | Official, integrates with Compose |
| Android ORM | Room | Official SQLite wrapper |
| HTML Parser | HtmlAgilityPack (.NET) / JSoup (Kotlin) | Battle-tested |
| Password Crypto | AES-256-GCM | Industry standard authenticated encryption |

## Verification Plan

After each phase, verify:

1. **Phase 1**: `dotnet build` succeeds for all .NET projects. WPF app launches, shows tab strip, can navigate to a URL via address bar.
2. **Phase 2**: Bookmarks can be added/edited/deleted via sidebar. History records visits. Settings persist across restart. Theme switching works.
3. **Phase 3**: PDF files open in-browser. Reading mode extracts clean content. Passwords save/load with encryption. DevTools opens.
4. **Phase 4**: Unpacked Chrome extensions load and function. Extension popup renders. Content scripts inject on page load.
5. **Phase 5**: Android app builds (`./gradlew assembleDebug`). WebView loads pages. Tabs, bookmarks, history work. Reading mode and PDF viewer functional.
